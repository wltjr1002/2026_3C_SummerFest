# 삼창 플레이어 원 PPT 아웃라인

## 1. 제작 목표

- 사용 목적: 20명 규모 멤버십 트레이닝 레크리에이션 진행용 PPT
- 단체명: 삼창
- 전체 테마: 레디 플레이어 원을 가볍게 패러디한 가상 세계 토너먼트
- 참가 방식: 4개 팀 경쟁
- PPT 톤: 진지한 SF보다는 게임 쇼, VR 아레나, 튜토리얼 안내창, 버그 난 시스템 메시지 느낌
- 최종 목표: 각 팀이 미니게임을 통해 능력치와 유물을 모아 최종 전투에서 승리

## 2. 핵심 진행 규칙

### 팀 구성

- 총 20명
- 4개 팀, 팀당 5명
- 팀명은 현장에서 정하거나 PPT에서 기본값 제공
  - 팀 A: 블루 게이트
  - 팀 B: 레드 포탈
  - 팀 C: 그린 코드
  - 팀 D: 옐로 버퍼

### 능력치

각 게임 승리 팀은 다른 세션에서 제작 중인 웹 게임에 반영할 능력치를 얻는다.

| 능력치 | 의미 | PPT 표현 |
| --- | --- | --- |
| 공격력 | 최종 전투에서 주는 피해 증가 | 검, 레이저, 크리티컬 효과 |
| 방어력 | 최종 전투에서 받는 피해 감소 | 방패, 배리어, 방어막 효과 |
| 운 | 랜덤 이벤트, 보너스 굴림, 동점 우선권 | 주사위, 글리치, 복권 효과 |
| 스피드 | 선공권, 제한시간 보너스, 순서 선택권 | 번개, 부스터, 대시 효과 |

### 포인트 지급 기준

- 일반 게임: 1위 3포인트, 2위 2포인트, 3위 1포인트, 4위 0포인트
- 짧은 보너스 라운드: 1위 2포인트, 2위 1포인트
- 팀 대결형 게임: 승리 팀 3포인트, 패배 팀 1포인트
- 유물 문제: 정답 팀이 유물 1개 획득. 능력치 포인트와 별도.

### 유물

각 소단원 마지막에 어려운 문제를 배치한다. 정답 팀은 유물을 획득한다.

유물 예시:

- 첫 번째 열쇠: 한 번만 오답을 무효 처리
- 관리자 토큰: 원하는 능력치 1포인트 추가
- 버그 조각: 상대 팀 하나의 최종 전투 선공권 무효
- 오아시스 지도: 최종 전투에서 랜덤 보너스 1회

## 4. 공통 디자인 가이드

### 비주얼 키워드

- VR 토너먼트
- 가상 세계 대기실
- 레트로 아케이드
- 네온 UI
- 게임 HUD
- 포털
- 랭킹 보드
- 시스템 알림창
- 글리치 효과

### 권장 이미지 검색어

저작권 문제가 적은 이미지가 필요하므로 Unsplash, Pexels, Pixabay, Wikimedia Commons, 공공누리, 직접 생성 이미지를 우선 사용한다.

- `arcade game neon background`
- `virtual reality headset people`
- `esports tournament stage`
- `retro arcade cabinet`
- `futuristic game interface`
- `neon portal background`
- `glitch texture transparent`
- `pixel game background`
- `sci fi dashboard ui`

### 생성형 이미지 프롬프트 공통 방향

상업용 행사 자료가 아니라 내부 레크리에이션용이라도, 특정 영화의 로고나 장면을 직접 복제하지 않는다. "레디 플레이어 원"이라는 문구 대신 다음처럼 분위기만 차용한다.

```text
A playful virtual reality tournament lobby for a Korean membership training event, neon arcade lights, retro game interface, four team portals, energetic but not serious, 16:9 presentation background, no text, no logos
```

### PPT 제작 스타일

- 배경: 어두운 네온 계열 또는 아케이드 배경 이미지
- 제목: 짧고 크게
- 진행 안내: 시스템 메시지처럼 표시
- 점수판: 4팀 능력치 표를 반복 사용
- 각 게임 시작 슬라이드: 게임명, 보상 능력치, 제한시간, 승리 조건
- 각 게임 종료 슬라이드: 능력치 획득 애니메이션, 랭킹 보드

## 5. 자료 폴더 계획

게임별 자료는 `resources/<game-name>/input/`에 원본, `resources/<game-name>/output/`에 가공 결과를 둔다.

초기화 명령 예시:

```powershell
python tools/file_processor.py speed-quiz --init
python tools/file_processor.py visible-overlap --init
python tools/file_processor.py bad-poster --init
python tools/file_processor.py art-style-guess --init
python tools/file_processor.py round-robin-battle --init
python tools/file_processor.py three-line-drawing --init
python tools/file_processor.py three-syllable-chain --init
python tools/file_processor.py beat-switch-songs --init
python tools/file_processor.py reversal-auction --init
python tools/file_processor.py tournament-battle --init
```

- 오답 시 "ACCESS DENIED" 팝업

## 8. 소단원 1: 기초 학습

세계관 설명: 가상 세계에 막 접속한 플레이어들이 기본 감각을 테스트받는 구간.

### 게임 1. 스피드 퀴즈

#### 게임 설명

특정 주제에 대한 빠른 문답 게임. 팀당 대표 1명이 제한시간 동안 최대한 많이 맞힌다.

#### 추천 주제

- 삼창 멤버 관련 가벼운 정보
- 여름, MT, 음식, 영화, K-POP, 밈
- 누구나 알 만한 상식

#### 진행 방식

- 팀당 60초
- 한 문제당 1점
- 모르면 패스 가능
- 동점이면 패스 수가 적은 팀 우선

#### 보상

- 1위: 스피드 +3
- 2위: 스피드 +2
- 3위: 스피드 +1

#### 슬라이드 구성

1. 게임 소개
2. 예시 문제
3. 팀별 문제 슬라이드 10~15장
4. 결과 입력 슬라이드
5. 능력치 획득 슬라이드

#### 필요한 자료

- 문제 텍스트
- 타이머 UI
- 정답/패스 효과음

#### 제작 프로그램

- PowerPoint 애니메이션: 타이머 바
- Excel 또는 Google Sheets: 문제와 정답 관리

### 게임 2. 안숨은그림찾기

#### 게임 설명

여러 연예인 또는 유명 인물 사진을 반투명하게 겹쳐 보여준다. 숨은그림찾기처럼 어렵게 숨기기보다, 좌우로 흩어 겹쳐서 알아볼 수 있게 만든다.

#### 진행 방식

- 라운드당 1개 합성 이미지 제시
- 제한시간 30초
- 팀별 화이트보드에 인물 이름 작성
- 한 이미지에 포함된 인물 수만큼 부분 점수 부여

#### 보상

- 1위: 운 +3
- 2위: 운 +2
- 3위: 운 +1

#### 슬라이드 구성

1. 게임 소개
2. 예시 합성 이미지
3. 문제 이미지 5~7장
4. 정답 공개 슬라이드
5. 능력치 획득 슬라이드

#### 이미지 검색 계획

초상권 이슈를 줄이려면 공개 사용 가능한 인물 사진보다 내부용으로 직접 만든 AI풍 실루엣 또는 진행자가 보유한 사용 가능한 이미지를 우선한다. 실제 연예인 사진을 사용할 경우 외부 배포하지 않는 내부 행사 자료로 제한한다.

검색어 예시:

- `Korean celebrity portrait transparent background`
- `famous actor portrait public domain`
- `kpop idol press photo`
- `celebrity face png`

#### 제작 프로그램

- PowerPoint: 투명도 조절, 좌우 배치
- Photopea 또는 Photoshop: 배경 제거, 투명도 적용
- Python 후보 프로그램: 여러 이미지를 자동으로 반투명 합성하는 스크립트

#### 제작할 프로그램

`tools/create_visible_overlap.py`

- 입력: `resources/visible-overlap/input/`의 인물 이미지
- 처리:
  - 이미지 크기 통일
  - 배경 제거된 PNG를 반투명 처리
  - x좌표를 조금씩 흩어 배치
  - 합성 결과와 정답표 생성
- 출력:
  - `resources/visible-overlap/output/round-01.png`
  - `resources/visible-overlap/output/answers.csv`

### 유물 문제 1. 첫 번째 열쇠

#### 문제 콘셉트

기초 라운드의 모든 정답 첫 글자를 조합하면 숨겨진 문장이 나온다.

#### 예시

- 스피드 퀴즈 정답 첫 글자와 안숨은그림찾기 인물 성을 모으면 `삼창서버입장완료`

#### 보상

- 유물: 첫 번째 열쇠
- 효과: 이후 게임에서 한 번만 오답 1개 무효

## 9. 소단원 2: 실전 훈련

세계관 설명: 플레이어가 가상 세계의 이미지 변조 구역에 입장한다.

### 게임 3. 대충그린 포스터

#### 게임 설명

유명 영화 포스터를 AI로 "못 그린 그림", "어설픈 어린이 그림", "기억만으로 그린 포스터" 느낌으로 변환한다. 팀은 원본 영화 제목을 맞힌다.

#### 진행 방식

- 문제당 제한시간 30초
- 정답 영화 제목 1점
- 힌트 공개 후 정답은 0.5점
- 8~10문제

#### 보상

- 1위: 공격력 +3
- 2위: 공격력 +2
- 3위: 공격력 +1

#### 슬라이드 구성

1. 게임 소개
2. 예시: 변환 이미지와 원본
3. 문제 슬라이드
4. 힌트 슬라이드
5. 정답 공개 슬라이드
6. 능력치 획득 슬라이드

#### 이미지 검색 계획

직접 포스터 원본을 PPT에 크게 싣기보다, 변환된 그림을 중심으로 사용한다. 정답 공개 슬라이드에는 영화 제목 텍스트와 작은 참고 이미지만 사용하거나 포스터 대신 키워드 이미지를 사용한다.

검색어 예시:

- `movie poster [영화명]`
- `minimal movie poster [영화명]`
- `iconic movie poster reference`
- `public domain movie poster`

추천 영화 후보:

- 기생충
- 범죄도시
- 부산행
- 어벤져스
- 해리포터
- 겨울왕국
- 타이타닉
- 라라랜드
- 인터스텔라
- 스즈메의 문단속

#### 생성형 이미지 프롬프트 예시

```text
A badly drawn childish crayon version of an iconic movie poster, intentionally awkward proportions, recognizable composition but no text, no logos, simple colors, funny, 16:9
```

#### 제작 프로그램

- 이미지 생성 도구: 포스터 변환 이미지 생성
- PowerPoint: 문제/힌트/정답 슬라이드 구성
- Excel 또는 CSV: 영화명, 힌트, 정답 관리

#### 제작할 프로그램

`tools/build_poster_round.py`

- 입력:
  - `resources/bad-poster/input/posters.csv`
  - 변환 이미지 파일
- 처리:
  - 문제 번호 부여
  - 이미지 파일명과 정답 매핑
  - 진행자용 정답표 생성
- 출력:
  - `resources/bad-poster/output/answer-sheet.csv`
  - `resources/bad-poster/output/slide-list.md`

### 게임 4. 그림체 맞히기

#### 게임 설명

인물을 특정 그림체로 변환한 이미지를 보여준다. 팀은 "그림체"와 "인물"을 모두 맞힌다.

#### 진행 방식

- 한 문제당 2점
  - 그림체 1점
  - 인물 1점
- 제한시간 40초
- 6~8문제

#### 보상

- 1위: 운 +3
- 2위: 운 +2
- 3위: 운 +1

#### 그림체 후보

저작권과 작가 스타일 직접 모방 논란을 줄이기 위해 생존 작가 이름이나 특정 스튜디오명을 피하고, 넓은 장르 표현을 사용한다.

- 90년대 일본 애니풍
- 픽셀 아트풍
- 웹툰 썸네일풍
- 미국 코믹북풍
- 르네상스 초상화풍
- 점토 애니메이션풍
- 초등학생 색연필 그림풍
- 게임 캐릭터 선택창풍

#### 인물 후보

- 참가자들이 모두 아는 내부 인물
- 유명 방송인
- 역사적 인물
- 영화 캐릭터

#### 슬라이드 구성

1. 게임 소개
2. 채점 예시
3. 문제 이미지
4. 정답 공개: 그림체 + 인물
5. 능력치 획득 슬라이드

#### 이미지 검색 계획

검색어 예시:

- `public domain portrait historical figure`
- `celebrity portrait reference`
- `Korean entertainer portrait`
- `pixel art portrait style reference`
- `comic book portrait style reference`

#### 생성형 이미지 프롬프트 예시

```text
Transform this person into a retro pixel art character selection portrait, vibrant colors, square composition, game UI feel, no text, no logos
```

#### 제작 프로그램

- 이미지 생성 도구: 인물 이미지 변환
- Photopea 또는 Photoshop: 얼굴 가림, 크롭, 색감 통일
- PowerPoint: 정답 애니메이션

#### 제작할 프로그램

`tools/build_art_style_round.py`

- 입력:
  - `resources/art-style-guess/input/rounds.csv`
  - 문제 이미지
- 처리:
  - 그림체명, 인물명, 파일명 매핑
  - 문제 순서 랜덤화

### 게임 5. 라운드 로빈 전투

#### 슬라이드 구성

3. 전투 규칙
4. 라운드 로빈 대진표
5. 경기별 전투 슬라이드 6장

- 팀별 답안 작성

#### 보상

- 3위: 공격력 +1

#### 슬라이드 구성

1. 게임 소개
2. 예시 믹스
3. 라운드별 재생 슬라이드
4. 정답 공개
5. 능력치 획득 슬라이드

준비 기준:

- 같은 BPM 또는 비슷한 BPM의 곡 3개 묶기
- 후렴구처럼 알아듣기 쉬운 구간 사용
- 너무 어려운 곡 1개, 쉬운 곡 2개 비율

검색어 예시:

- `[song title] bpm`
- `K-pop songs 120 bpm`
- `popular Korean songs same bpm`
- `BPM analyzer online`

#### 제작 프로그램

### 게임 11. 토너먼트 전투

#### 진행 방식

- 4강 2경기, 결승 1경기, 3/4위전 선택
- 대진은 중간 평가 순위 또는 현재 총 능력치 순위 기준
  - 1위 vs 4위
  - 2위 vs 3위
- 각 팀 기본 HP: 10
- 공격 턴:
  - 피해량 = 1 + 공격력 보너스 + 주사위 보정
  - 방어팀은 방어력에 따라 피해 0~2 감소
- 스피드가 높은 팀이 선공
- 운이 높은 팀은 경기당 1회 재굴림 가능
- 유물은 각 효과에 맞게 1회 사용

#### 간단 계산안

- 공격력 0~2: 추가 피해 0
- 공격력 3~5: 추가 피해 1
- 공격력 6 이상: 추가 피해 2
- 방어력 0~2: 피해 감소 0
- 방어력 3~5: 피해 감소 1
- 방어력 6 이상: 피해 감소 2
- 스피드가 높으면 선공
- 운 3 이상: 경기당 재굴림 1회
- 운 6 이상: 경기당 재굴림 2회

#### 슬라이드 구성

1. 최종 전투 진입
2. 최종 능력치 보드
3. 유물 보유 현황
4. 4강 대진표
5. 4강 1경기 전투 슬라이드
6. 4강 2경기 전투 슬라이드
7. 결승 진출 연출
8. 결승 전투 슬라이드
9. 최종 우승 발표
10. 엔딩 크레딧

#### 필요한 자료

- 토너먼트 대진표
- HP 바
- 공격/방어/운/스피드 아이콘
- 유물 카드
- 우승 배경

#### 제작 프로그램

- PowerPoint: 대진표, HP 바, 전투 애니메이션
- Excel 또는 Google Sheets: 전투 계산
- 웹 게임 세션과 연동 시: 최종 능력치 값을 웹 게임 입력값으로 전달

## 17. 리허설 체크

- 20명이 이해하는 데 3분 이상 걸리는 규칙은 줄인다.
- 각 게임은 설명 포함 8~15분 안에 끝나도록 조정한다.
- 노래 게임은 음량과 재생 장치를 사전 점검한다.
- 이미지 맞히기 게임은 너무 어둡거나 작지 않게 테스트한다.
- 최종 전투 계산은 진행자가 실수하지 않도록 별도 계산표를 사용한다.
- 유물 효과는 강하지만 승부를 완전히 망가뜨리지 않게 1회성으로 제한한다.
